      Security and transparency

      Built like a careful product, not a loud promise.

      What this MVP does

      • Connects to injected wallets like MetaMask
      • Runs client-side demo analytics
      • Stores simulated claims in browser local storage
      • Deploys directly on GitHub Pages

      What this MVP does not do

      • No real gas refunds from the blockchain
      • No backend or partner rebate engine yet
      • No private identity scraping
      • No custody of user funds
      FAQ

      Clear answers for users and partners.

      Is this a real gas refund product?

      This version is a frontend MVP. It simulates eligibility, recovery value, and claim flows to demonstrate the product experience.

      Does it work on GitHub Pages?

      Yes. It is a plain static site with HTML, CSS, and JavaScript, so it can go live directly on GitHub Pages without a build tool.

      Does it read private wallet data?

      No. The concept is positioned around public on-chain signals and optional user-authorized linking only.

      What should be built next?

      A real backend, on-chain indexing, claim funding logic, supported chain integrations, and a proper security review.
